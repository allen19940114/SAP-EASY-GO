## 1. 💬 AI 智能对话

**位置**: 首页 `/`

### 功能说明

与OLORA AI助手进行自然语言对话，处理各类SAP业务查询。

### 如何使用

1. **快速开始**: 点击预设问题卡片，快速体验对话功能
2. **输入问题**: 在底部输入框输入您的问题
3. **发送消息**: 按Enter键或点击"发送"按钮
4. **查看回复**: AI助手会实时给出专业回复

### 支持的场景

- 📊 SAP项目创建与管理
- 💰 预算查询与分析
- 📝 业务报表生成
- 📚 知识库检索
- 🔒 数据安全咨询
- ⚙️ 系统功能介绍

### 示例对话

```
用户: 如何创建一个SAP项目？
AI: 我可以帮您创建SAP项目。请提供以下信息：
    - 项目名称
    - 预算金额
    - 负责人
    - 开始日期

    💡 示例：
    "创建一个名为'华为5G基站'的项目，预算500万元，
     负责人张三，2025年1月开始"
```

## 2. 📚 知识库管理

**位置**: `/knowledge`

### 功能说明

上传和管理业务文档，支持智能检索和问答。

### 核心功能

#### 📤 文档上传
- 支持格式：PDF、Word、TXT
- 自动解析文档内容
- 分块向量化存储
- 存储到Qdrant向量数据库

#### 📋 文档列表
- 查看所有已上传文档
- 显示文档大小、上传时间、处理状态
- 支持查看和删除操作

#### 🔍 语义搜索
- 基于向量相似度检索
- 自动找到最相关的文档片段
- 在AI对话中自动引用

### 如何配置

1. **启动Qdrant向量数据库**:
   ```bash
   docker run -d --name qdrant -p 6333:6333 qdrant/qdrant
   ```

2. **配置连接**: 在"系统设置"页面配置Qdrant URL

3. **上传文档**: 点击"📤 上传文档"按钮选择文件

### 工作流程

```
文档上传 → 解析内容 → 文本分块 → 向量化 → 存储到Qdrant
                                              ↓
用户提问 → 向量检索 → 找到相关片段 → 注入上下文 → AI回答
```

## 3. ⚙️ 系统设置

**位置**: `/settings`

### 功能说明

配置LLM模型、向量数据库等核心参数。

### 3.1 大语言模型配置

#### 支持的模型提供商

| 提供商 | 模型 | 获取API Key |
|--------|------|-------------|
| **OpenAI** | GPT-4, GPT-3.5 | https://platform.openai.com/api-keys |
| **DeepSeek** | DeepSeek Chat | https://platform.deepseek.com |
| **Google** | Gemini Pro | https://makersuite.google.com/app/apikey |

#### 配置步骤

1. **选择模型提供商**: 从下拉菜单选择（OpenAI/DeepSeek/Gemini）

2. **输入API Key**:
   ```
   OpenAI: sk-...
   DeepSeek: sk-...
   Gemini: AI...
   ```

3. **调整模型参数**:
   - **Temperature**: 0-2之间，控制创造性（推荐0.7）
   - **Max Tokens**: 最大生成长度（推荐2000）

4. **保存配置**: 点击"💾 保存配置"

5. **重启后端**: 配置生效需要重启后端服务
   ```bash
   # 停止旧进程
   lsof -ti:3002 | xargs kill

   # 重新启动
   cd apps/backend
   node src/standalone-api.ts
   ```

### 3.2 向量数据库配置

#### Qdrant配置

1. **服务地址**:
   - 默认: `http://localhost:6333`
   - Docker启动命令已在页面提供

2. **Collection名称**:
   - 默认: `olora_knowledge`
   - 自动创建，无需手动设置

3. **Embedding模型**:
   - `text-embedding-ada-002` (推荐)
   - `text-embedding-3-small`
   - `text-embedding-3-large`

### 配置文件位置

所有配置保存到：`apps/backend/.env`

```env
# LLM配置
OPENAI_API_KEY=sk-your-api-key
DEEPSEEK_API_KEY=sk-your-api-key
GEMINI_API_KEY=AI-your-api-key

# 向量数据库
QDRANT_URL=http://localhost:6333
QDRANT_COLLECTION=olora_knowledge

# 模型参数
LLM_TEMPERATURE=0.7
LLM_MAX_TOKENS=2000
```

## 🔧 完整配置流程

### 从零开始配置OLORA

#### 步骤1: 启动基础服务

```bash
# 1. PostgreSQL（已启动）
# 2. 前端（已启动 - 端口3001）
# 3. 后端API（已启动 - 端口3002）
```

#### 步骤2: 配置LLM（可选但推荐）

1. 访问 http://localhost:3001/settings
2. 选择LLM提供商（推荐OpenAI）
3. 输入API Key
4. 保存配置
5. 重启后端服务

#### 步骤3: 启动向量数据库（可选）

```bash
docker run -d \
  --name qdrant \
  -p 6333:6333 \
  -v $(pwd)/qdrant_storage:/qdrant/storage \
  qdrant/qdrant
```

#### 步骤4: 上传知识库文档（可选）

1. 访问 http://localhost:3001/knowledge
2. 点击"📤 上传文档"
3. 选择PDF/Word/TXT文件
4. 等待处理完成

#### 步骤5: 开始使用

1. 返回首页: http://localhost:3001
2. 输入问题，开始对话
3. AI会自动检索知识库提供精准回答

## 💡 使用技巧

### 1. 充分利用预设问题

首页的4个预设问题卡片可以快速开始对话，了解系统功能。

### 2. 结构化提问

良好的提问方式：
```
❌ 不好: "项目"
✅ 良好: "如何创建一个SAP项目？"
✅ 更好: "创建一个名为'华为5G'的项目，预算500万"
```

### 3. 上传高质量文档

- ✅ 结构清晰的PDF文档
- ✅ 格式规范的Word文档
- ✅ 业务相关的专业内容
- ❌ 避免扫描图片PDF
- ❌ 避免格式混乱的文档

### 4. 定期检查审计日志

建议每周查看审计日志，确保：
- 操作符合预期
- 无异常访问
- 数据安全可控

## 📞 获取帮助

### 遇到问题？

1. **查看文档**: README.md, USAGE_GUIDE.md
2. **检查日志**: 查看后端服务日志
3. **重启服务**: 尝试重启前后端
4. **查看审计日志**: 确认操作是否被记录

### 常见问题

**Q: AI回复不准确？**
A: 配置真实的LLM API Key（在系统设置页面）

**Q: 知识库检索不到内容？**
A: 确保Qdrant已启动，并上传了相关文档

**Q: 页面无法访问？**
A: 检查前端服务（端口3001）和后端服务（端口3002）是否运行
